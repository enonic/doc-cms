Content is the core entity in Enonic CMS. Every content item has a specific content type and life cycle.

== Intro
Content can be created, edited, published and deleted via Content Studio, or manipulated programmatically. Every Content item has a specific <<content/content-type#, content type>>, which defines its data structure and capabilities. Content items are stored in <<projects#, content projects>>, where each item is indexed and can be searched for and accessed via API.

== Content authoring




== Anatomy

Content items can be accessed via the GraphQL API, which can be used by front-end developers to access content programmatically. The GraphQL API is generated based on the content type's schema and form definition, and provides a powerful and flexible way to query for content items and their properties.
