= Form Items
:toc: right
:imagesdir: images

Form items are the building blocks of forms. They define structure and content of a form. 

== Introduction

A form item will produce output based on its type and configuration. Each form item has a front-end and back-end component. The front-end represents the visual editing interface used in the XP admin console, where the back-end is solely used for validation purposes.

A form item typically produces a property, with a specific https://developer.enonic.com/docs/xp/stable/storage/indexing[value type], with zero, 1 or multiple occurences. For instance, a `TextLine` will always produce a `String` value, while the output of an `ItemSet` is an object AKA propertySet. 

== Usage

image::default-input.png[An input box with help-text below it, 650w]

The following configuration is common for for most form items:

[source,yaml]
----
- type: "InputTypeName"  # <1>
  name: "my_name"
  label:  "My label" # <2>
  default: ""  # <3>
  help-text: "Help text here" # <4>
  occurrences:  # <5>
    min: 0
    max: 1
  # config properties are specified directly  # <6>
----

<1> *type* and *name* are mandatory: +
_name_ is used when storing the data in a property, and must be unique on each level. +
_type_ refers to one of the many input types which are listed below.
+
NOTE: Use `snake_case` (lowercase with underscores) for all names. Capital letters are flattened during indexing which can cause unexpected query behavior. The `snake_case` convention also ensures clean field names in the <<../apis/graphql#, GraphQL API>>.
+
[source,yaml]
----
name: "my_field_name"    # Recommended
name: "myFieldName"      # Avoid - capitals are flattened in the index
name: "myfieldname"      # Avoid - hard to read
----

<2> *label* is another mandatory field that holds the human readable value that will be displayed when listing the input type control in the administrative interface.
 
<3> *default* is an optional field that lets you specify default values to be used by the input type.

<4> *help-text* is an optional field that lets you specify a text label shown below the input field. Used for explanation of the field's purpose. +
Like `label`, supports <<i18n#,localization>> via the `text`/`i18n` object pattern.

<5> *occurrences* is an optional field used to control the number of values stored by a single input. +
_min_ set to zero means the input is not mandatory +
_max_ set to zero means there is no upper limit to the number of values. +
This element is optional, if omitted the default will be `min: 0` and `max: 1`.

<6> Config properties are specified directly on the form item, rather than in a separate config element.

== Form items grouped by output

Each form item produces a value of a specific https://developer.enonic.com/docs/xp/stable/storage/indexing#value_types[value type]. The value type determines how the data is stored and indexed.

=== String
* <<textline.adoc,TextLine>> — single line plain text
* <<textarea.adoc,TextArea>> — multi-line plain text
* <<htmlarea.adoc,HtmlArea>> — multi-line rich text (HTML)
* <<combobox.adoc,ComboBox>> — select from predefined options
* <<radiobutton.adoc,RadioButton>> — single select from predefined options
* <<tag.adoc,Tag>> — free-text tags
* <<customselector.adoc,CustomSelector>> — custom data source selection
* <<contenttypefilter.adoc,ContentTypeFilter>> — content type name

=== Reference
* <<contentselector.adoc,ContentSelector>> — reference to other content
* <<imageselector.adoc,ImageSelector>> — reference to image content
* <<mediaselector.adoc,MediaSelector>> — reference to media content

=== Attachment Reference
* <<attachmentuploader.adoc,AttachmentUploader>> — file attachment

=== Long
* <<long.adoc,Long>> — 64-bit integer number

=== Double
* <<double.adoc,Double>> — 64-bit floating-point number

=== Boolean
* <<checkbox.adoc,CheckBox>> — true/false value

=== GeoPoint
* <<geopoint.adoc,GeoPoint>> — latitude/longitude coordinate

=== LocalDate
* <<date.adoc,Date>> — date without time

=== LocalTime
* <<time.adoc,Time>> — time without date

=== Instant
* <<instant.adoc,Instant>> — date and time without timezone

=== DateTime
* <<datetime.adoc,DateTime>> — date and time with timezone

=== Set (PropertySet)
* <<item-set.adoc,ItemSet>> — nested group of form items
* <<option-set.adoc,OptionSet>> — selectable groups of form items

=== No output
* <<field-set.adoc,FieldSet>> — visual grouping only
