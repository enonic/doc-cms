= CustomSelector input type

:imagesdir: images
:toc: right

Selector with customizable data source. Useful for live integrations with external data sources.

TIP: Tutorial: Check out https://developer.enonic.com/docs/build-a-custom-selector[Building a custom selector] for hands on learning.

== Appearance

image::Custom-selector.png[Custom selector dropdown with options from an external service, 700]

== Usage

The `service` attribute points to an HTTP endpoint (a function provided by an Enonic XP app) that returns matching options as JSON. See the Enonic XP app-development docs for how to implement that function.

Below are several sample usages of CustomSelector:

[source,yaml]
----
# Basic Sample
- type: "CustomSelector"
  name: "my_custom_selector"
  label: "My Custom Selector"
  occurrences:
    min: 0
    max: 0
  service: "my-custom-selector"  # <1>

# Sample with parameters
- type: "CustomSelector"
  name: "music_track"
  label: "Intro song"
  service: "spotify-music-selector"
  param:  # <2>
    - value: "genre"
      label: "classic"
    - value: "sortBy"
      label: "length"

# Gallery mode
- type: "CustomSelector"
  name: "album_cover"
  label: "Album cover"
  service: "spotify-album-cover-selector"
  galleryMode: true  # <3>
----
<1> *service* — the name of the service function that supplies options. Can reference a function in another application using the `com.myapplication.app:myservice` form.
<2> *param* lets you pass parameters to the service so the same function can be reused across fields. Parameters are sent as query-string name/value pairs.
<3> *galleryMode* set to `true` renders options in a gallery style (three big images in a row), similar to <<imageselector#appearance,ImageSelector>>. Useful when images should be emphasized.

=== Option JSON format

The service function returns JSON with `total`, `count`, and a `hits` array. Each hit becomes a selectable option:

[source,JSON]
----
{
  "total": 10,
  "count": 2,
  "hits": [
    {
      "id": "1",
      "displayName": "Option number 1",
      "description": "Short description shown under the title",
      "iconUrl": "/some/path/images/number_1.svg"
    },
    {
      "id": "2",
      "displayName": "Option number 2",
      "description": "Inline SVG markup is used as icon",
      "icon": {
        "data": "<svg xmlns=\"http://www.w3.org/2000/svg\"/>",
        "type": "image/svg+xml"
      }
    }
  ]
}
----

Each hit requires `id` and `displayName`. `description`, `iconUrl`, and inline `icon` are optional. The `id` is what gets stored when the option is selected — see <<#output,Output>>.

For how to implement the service function (request parameters, pagination, filtering), see the Enonic XP app-development docs.

== Output

*Value type:* `String` — not included in `_alltext`

The stored value is the `id` of the selected item. When multiple selections are allowed, the value is an array.

[source,JSON]
----
{
  "my_custom_selector": "1"
}
----

.Multiple selections
[source,JSON]
----
{
  "my_custom_selector": [
    "1",
    "2"
  ]
}
----
